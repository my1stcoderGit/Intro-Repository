Git & GitHub True WORKFLOW

1. Create a local repository on your machine wite your code and intialize it by using git init command.
2. Create a new repository on GitHub with the same name as local repo and copy its url.
3. Now link both local and remote repo using git remote add origin .
4. Then verify the linkage using git remote -v command.
5. If you accidently link to wrong remote repo then you can remove it using git remote remove origin command and then link it again using git remote add origin .
6. Then apply git add . command. Donot run git add . command in an empty folder atleast create one html dummy file to avoid errors while pushing to remote.
7. Then apply commit command git commit -m "message".
8. Then forcely push the changes using git push origin --force command.

False Workflow

1. Create a new repository on GitHub.
2. Create a local repository on your machine and intialize it by using git init command.
2. Clone the repository to your local machine using git clone [repository URL].
3. Navigate to the repository directory using cd [repository name].
4. Make changes to your files and use git add [file name] or git add . to stage the changes.
5. Commit the changes with a descriptive message using git commit -m "your message".
6. Push the changes to the remote repository using git push origin [branch name].
